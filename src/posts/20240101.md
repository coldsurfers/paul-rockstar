# IntersectionObserver — 스크롤 감지의 올바른 방법

> 요소가 뷰포트에 들어왔는지 비동기로 감지하는 웹 API.

---

## 왜 scroll 이벤트 대신인가

전통적으로 스크롤을 감지할 때 `window scroll` 이벤트 + `addEventListener`를 썼다. 하지만 두 가지 문제가 있다.

1. 콜백이 과도하게 호출된다 — 디바운스/스로틀로 완화할 수 있지만 근본적인 해결은 아니다.
2. 특정 영역을 감지하려면 `getBoundingClientRect()`를 써야 한다 — 호출할 때마다 reflow가 발생한다.

`IntersectionObserver`는 이 두 문제를 모두 피한다. 비동기로 동작하고, reflow 없이 교차 상태를 감지한다.

---

## 기본 구조

```js
const observer = new IntersectionObserver(callback, options);
```

### callback

```js
(entries, observer) => {
  entries.forEach(entry => {
    entry.isIntersecting  // 뷰포트와 교차 중인가
    entry.intersectionRatio  // 얼마나 겹쳤는가 (0.0 ~ 1.0)
    entry.target  // 감시 중인 요소
  });
}
```

### options

```js
{
  root: null,           // 기준 요소 (null이면 viewport)
  rootMargin: '0px',    // 기준 영역 확장/축소 (CSS margin과 동일)
  threshold: 1.0,       // 교차 비율 임계값 (1.0 = 100% 가시)
}
```

### observer 메서드

```js
observer.observe(target)    // 감시 시작
observer.unobserve(target)  // 특정 요소 감시 중지
observer.disconnect()       // 전체 감시 중지
```

## 실전 예시

### 무한 스크롤

리스트 하단에 로딩 인디케이터 div를 두고, 그것이 뷰포트에 들어오면 다음 페이지를 불러온다.

```tsx
useEffect(() => {
  const observer = new IntersectionObserver(
    (entries) => {
      const [entry] = entries;
      if (entry.isIntersecting) {
        loadMore();
      }
    },
    { threshold: 0.5 }
  );

  if (loadingIndicatorRef.current) {
    observer.observe(loadingIndicatorRef.current);
  }

  return () => observer.disconnect();
}, [articles.length, loadMore]);
```

### 동적 고정 사이드바

특정 높이의 sentinel div를 감시하다가, 뷰포트 밖으로 나가는 순간 사이드바를 `fixed`로 전환한다.

```tsx
useEffect(() => {
  const observer = new IntersectionObserver(
    ([entry]) => {
      setIsFixed(!entry.isIntersecting);
    },
    { threshold: 0.1 }
  );

  if (sentinelRef.current) {
    observer.observe(sentinelRef.current);
  }

  return () => observer.disconnect();
}, []);
```

```tsx
const Sidebar = styled.ul<{ isFixed: boolean }>`
  position: ${(p) => (p.isFixed ? 'fixed' : 'relative')};
  top: ${(p) => (p.isFixed ? '1rem' : '0px')};
`;
```
