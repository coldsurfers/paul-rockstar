# 공공 API를 CLI로 감싸기 — kopis-cli 3일간의 빌드업

> "데이터는 이미 열려 있다. 문제는 그걸 쓸 수 있는 형태로 만드는 사람이 없다는 것이다."

---

KOPIS(공연예술통합전산망)는 한국 공연 정보의 거의 유일한 공공 데이터 소스다. 공연 목록, 상세 정보, 공연시설, 축제, 수상작, 기획사까지 — API는 열려 있다.

그런데 매번 Postman을 열고, XML 응답을 눈으로 파싱하고, 파라미터를 문서에서 찾아 복붙하는 건 생산적이지 않았다. COLDSURF 서비스의 공연 데이터 파이프라인을 설계하면서 KOPIS API를 반복적으로 호출할 일이 많아졌고, 이걸 터미널에서 바로 쓸 수 있는 도구로 만들기로 했다.

---

## Phase 1–2: 스캐폴딩과 API 클라이언트

첫날은 구조를 잡는 데 썼다. pnpm workspace로 모노레포를 구성하고, `@coldsurf/tickets`라는 이름으로 CLI 패키지를 스캐폴딩했다. KOPIS API의 XML 응답을 TypeScript 타입으로 정의하고, API 클라이언트를 먼저 만들었다.

공공 API 특유의 문제가 있다. 응답이 XML이고, 필드명이 한글 약자(`prfnm`, `prfplcnm`, `prfstate`)다. 이걸 그대로 노출하면 CLI를 쓰는 사람이 매번 문서를 뒤져야 한다. 타입 정의 단계에서 이 매핑을 확실히 잡아두는 게 핵심이었다.

---

## Phase 3–4: 커맨드 구현과 포맷터

`find`와 `detail` 커맨드를 구현하고, table/json 두 가지 출력 포맷을 지원하게 만들었다. 터미널에서 바로 테이블로 보는 게 기본이고, 파이프라인에 태울 때는 `--format json`으로 전환하면 된다.

필터 옵션을 하나씩 붙여나갔다. 날짜 범위, 카테고리, 지역/구군, 공연 상태, 어린이 공연 여부, 오픈런 여부, 공연장 코드, 제목/장소 검색까지. KOPIS API가 지원하는 파라미터를 빠짐없이 CLI 옵션으로 매핑하는 걸 목표로 했다. spec 체크리스트를 만들어두고 하나씩 체크하며 진행했다.

---

## Phase 5: npm 배포 파이프라인

코드가 동작하는 것과 남이 쓸 수 있는 것은 다른 문제다. changesets로 버전 관리를 세팅하고, GitHub Actions로 자동 퍼블리시 파이프라인을 구성했다. OIDC trusted publishing으로 npm provenance까지 붙였다.

`npx @coldsurf/tickets find --startDate 20250101` — 설치 없이 바로 실행할 수 있는 상태가 됐다.

---

## 확장: venue, festival, creator

기본 뼈대가 잡힌 뒤로는 도메인을 확장했다. 공연시설 목록/상세 조회(`venue`, `venue-detail`), 기획/제작사 목록(`producer`), 수상작 목록(`award`), 축제 목록(`festival`), 원작자 목록(`creator`)까지.

각 도메인마다 API 응답 구조가 조금씩 달랐지만, 첫 설계에서 API 클라이언트와 커맨드 구조를 분리해둔 덕에 새 커맨드 추가는 반복 작업에 가까웠다. 구조가 맞으면 확장은 빠르다.

---

## 배운 것

**공공 API는 래핑 한 겹이면 쓸 만해진다.** XML 응답, 암호 같은 필드명, 부실한 에러 메시지 — 이런 것들이 진입 장벽이지, API 자체가 나쁜 건 아니다. 얇은 추상화 레이어 하나만 씌우면 생산성이 완전히 달라진다.

**Phase 단위로 끊어야 속도가 난다.** 스캐폴딩 → API 클라이언트 → CLI 커맨드 → 포맷터 → 배포. 각 단계가 독립적으로 완결되니까 중간에 방향을 틀어도 앞 단계가 흔들리지 않았다. 3일 안에 npm 배포까지 간 건 이 구조 덕이다.

**도구를 만드는 건 도메인을 이해하는 가장 빠른 방법이다.** KOPIS API의 파라미터를 하나씩 CLI 옵션으로 옮기면서, 공연 데이터의 구조 — 공연/시설/기획사/축제/수상작이 어떻게 연결되는지 — 를 체감할 수 있었다. COLDSURF의 데이터 모델을 설계할 때 이 경험이 직접적으로 도움이 됐다.

---

> CLI는 가장 솔직한 인터페이스다. 필요한 것만 입력하고, 필요한 것만 출력한다. 그 사이에 군더더기가 없다.
