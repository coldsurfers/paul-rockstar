# 토큰이란 무엇인가 — LLM이 텍스트를 읽는 단위

> "GPT가 글자를 하나씩 읽는다고 생각했다면, 그건 반만 맞다."

---

LLM을 쓰다 보면 "토큰"이라는 단어를 자주 만난다. API 과금 기준이 토큰이고, 컨텍스트 윈도우 크기도 토큰으로 표시된다. 그런데 토큰이 정확히 뭔지 모른 채 넘어가는 경우가 많다.

---

## 토큰은 글자가 아니다

토큰은 알파벳 한 글자도 아니고, 단어 한 개도 아니다. LLM이 텍스트를 처리할 때 사용하는 **최소 단위**다.

모델은 텍스트를 있는 그대로 읽지 않는다. 입력된 텍스트를 먼저 **토크나이저(tokenizer)**라는 전처리기를 통해 토큰 시퀀스로 변환한 뒤 처리한다. 이 과정을 토큰화(tokenization)라고 한다.

---

## 어떻게 나뉘는가 — BPE

대부분의 최신 LLM은 **BPE(Byte Pair Encoding)** 알고리즘을 사용한다.

원리는 단순하다.

1. 텍스트를 바이트(또는 문자) 단위로 쪼갠다.
2. 가장 자주 붙어 나오는 쌍을 찾는다.
3. 그 쌍을 하나의 새로운 토큰으로 합친다.
4. 원하는 어휘 크기(vocabulary size)에 도달할 때까지 2~3을 반복한다.

결과적으로, 자주 쓰이는 단어는 통째로 하나의 토큰이 되고, 드문 단어는 여러 토큰으로 쪼개진다.

---

## 영어와 한국어의 차이

토큰화 효율은 언어에 따라 크게 달라진다.

### 영어

영어는 BPE 학습 데이터에서 압도적인 비중을 차지하기 때문에 효율이 높다.

| 텍스트 | 토큰 수 |
|--------|---------|
| `"a"` | 1 |
| `"apple"` | 1 |
| `"Hello, world!"` | 4 |
| `"The quick brown fox"` | 4 |

대략 **1 토큰 ≈ 영어 4글자** 또는 **0.75 단어**로 추정할 수 있다.

### 한국어

한국어는 사정이 다르다. 한글은 자모 조합으로 이루어진 문자 체계이고, BPE 학습 데이터에서의 비중이 영어보다 훨씬 작다. 그래서 같은 의미를 전달하더라도 더 많은 토큰을 소비한다.

| 텍스트 | 토큰 수 |
|--------|---------|
| `"안녕"` | 2~3 |
| `"안녕하세요"` | 3~5 |
| `"인공지능"` | 3~4 |

대략 **1 토큰 ≈ 한국어 1~2글자** 수준이다. 영어 대비 2~3배 비효율적이라고 보면 된다.

이건 비용에도 직접적으로 영향을 준다. 같은 내용을 한국어로 쓰면 토큰 소비가 늘어나고, API 비용도 그만큼 올라간다.

---

## 왜 토큰 단위인가

글자나 단어가 아닌 토큰을 쓰는 이유는 균형 때문이다.

- **글자 단위**: 어휘가 작아서 메모리는 적게 들지만, 시퀀스가 길어져서 연산 비용이 폭증한다.
- **단어 단위**: 시퀀스는 짧지만, 어휘가 너무 커서 모델 크기가 감당이 안 된다. 신조어나 오타도 처리 불가.
- **토큰 단위(BPE)**: 어휘 크기와 시퀀스 길이 사이에서 최적의 균형점을 찾는다.

---

## 실전에서 알아두면 좋은 것

### API 과금

OpenAI 기준으로 과금은 입력 토큰 + 출력 토큰의 합으로 계산된다.

```
비용 = (입력 토큰 × 입력 단가) + (출력 토큰 × 출력 단가)
```

예를 들어, `gpt-4o-mini`는 입력 $0.15/1M 토큰, 출력 $0.60/1M 토큰이다. 1M은 백만이니까, 일상적인 테스트에서는 사실상 무시할 수 있는 수준이다.

### 컨텍스트 윈도우

모델이 한 번에 처리할 수 있는 최대 토큰 수다. GPT-4o는 128K 토큰, Claude는 200K~1M 토큰을 지원한다. 입력과 출력을 합쳐서 이 한도 안에 들어와야 한다.

### 토큰 수 확인

OpenAI의 [tiktoken](https://github.com/openai/tiktoken) 라이브러리를 쓰면 텍스트가 몇 토큰인지 정확히 확인할 수 있다.

```python
import tiktoken

enc = tiktoken.encoding_for_model("gpt-4o")
tokens = enc.encode("Hello, world!")
print(len(tokens))  # 4
```

---

## 정리

- 토큰은 글자도, 단어도 아닌 LLM의 텍스트 처리 최소 단위다.
- BPE 알고리즘으로 자주 쓰이는 패턴을 하나의 토큰으로 합친다.
- 영어는 효율적이고, 한국어는 상대적으로 비효율적이다.
- API 비용, 컨텍스트 윈도우 전부 토큰 기준이므로 개념을 정확히 아는 게 중요하다.
